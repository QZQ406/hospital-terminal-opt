# 医院终端系统优化工具

专为医疗业务场景设计的轻量级系统优化工具，严格遵循零侵入、高稳定、数据安全、外设兼容、权限可控的核心原则。

**核心特性：**
- ✓ 单个exe文件（< 10MB）
- ✓ 免安装绿色软件
- ✓ U盘便携运行
- ✓ 前台GUI + 后台静默双模式
- ✓ 完整审计追踪

## 快速开始

### 系统要求
- Windows 7 SP1 或更高版本
- .NET Framework 4.6.2+
- 管理员权限（部分操作需要）

### 获取应用

直接下载单文件executable：`HospitalTerminalOpt.exe`

### 首次运行

1. **GUI模式**（前台运维）
   ```bash
   HospitalTerminalOpt.exe
   ```
   - 系统自动创建 `config/` 和 `logs/` 目录
   - 自动生成默认配置文件
   - 显示用户界面进行手动操作

2. **静默模式**（后台自动）
   ```bash
   HospitalTerminalOpt.exe -silent -optimize quick
   ```
   - 无UI显示
   - 自动执行优化操作
   - 结果记录到审计日志

### 配置域组（可选，用于权限管理）

由IT管理员创建以下Windows域组：
- `HOSPITAL\IT-ADMIN` - 运维人员（完全权限）
- `HOSPITAL\医护人员` - 医护人员（仅查看权限）

## 业务高峰保护

### 保护时段
**7:00 AM - 10:00 PM**（业务时间）

### 保护措施

| 时段 | 深度优化 | 打印服务重启 | 网络重置 | 极速优化 |
|------|--------|-----------|--------|--------|
| 业务高峰(7-22) | ✗ 禁用 | ✗ 禁用 | ✗ 禁用 | ✓ 可用 |
| 低峰时段(22-7) | ✓ 启用 | ✓ 启用 | ✓ 启用 | ✓ 可用 |

### 工作原理
1. **启动时检查**: 系统自动检查当前时间
2. **动态按钮控制**: 高峰时段禁用深度优化按钮
3. **业务感知**: 打印服务在高峰仅清理队列，不重启
4. **日志提示**: 用户可清楚看到为何某操作被禁用

## 禁止项配置

系统配置中严格禁止以下操作:

```xml
<AutoRestart enabled="false" reason="禁止自动重启" />
<ForcedUpdate enabled="false" reason="禁止强制更新" />
<BackgroundPopups enabled="false" reason="禁止后台弹窗" />
```

这确保:
- ✗ 系统不会在业务时间自动重启
- ✗ 系统不会强制安装更新
- ✗ 系统不会弹出中断业务的通知

## 故障排除

### 问题1: "权限拒绝"提示
**原因**: 用户不在IT-ADMIN组中
**解决**: 
1. 联系IT管理员
2. 将用户添加到`HOSPITAL\IT-ADMIN`组
3. 重新登录或重启应用

### 问题2: 深度优化按钮被禁用
**原因**: 当前在业务高峰时段(7:00-22:00)
**解决**: 
1. 正常情况（保护业务）
2. 等待低峰时段(22:00+)再执行
3. 或执行"极速优化"替代

### 问题3: 打印机不可用
**解决步骤**:
1. 点击"设备检查"确认驱动状态
2. 点击"打印修复"修复打印服务
3. 如仍不可用，检查打印机电源和连接

### 问题4: 医疗设备显示异常
**解决步骤**:
1. 点击"设备检查"查看具体错误码
2. 检查设备物理连接
3. 重启设备或联系供应商
4. 系统自动保护不卸载驱动

### 问题5: 找不到配置文件
**位置**: `%APPDATA%\HospitalTerminalOpt\config.xml`
**手动找到方法**:
1. Win+R，输入: `%APPDATA%`
2. 打开: `HospitalTerminalOpt` 文件夹
3. 打开: `config.xml` 编辑

## 配置管理

### 高级配置

编辑 `config.xml` 可自定义:

```xml
<!-- 业务高峰时段 -->
<BusinessPeakHours startHour="7" endHour="22" disableDeepOpt="true"/>

<!-- 优化时间限制 -->
<OptimizationPolicy>
  <QuickOpt maxDurationSeconds="300" allowDuringPeakHours="true"/>
  <DeepOpt maxDurationSeconds="1800" allowDuringPeakHours="false"/>
</OptimizationPolicy>

<!-- 日志保留 -->
<NotificationPolicy logRetentionDays="90"/>
```

### 配置修改建议
- 业务高峰时段: 根据医院实际调整
- 优化时间限制: 根据系统性能调整
- 日志保留天数: 根据合规要求调整
